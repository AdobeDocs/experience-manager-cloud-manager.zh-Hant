# 自訂許可權 {#custom-permissions}

了解如何使用自訂權限建立新的自訂權限設定檔並內含可設定的權限，以便限制 Cloud Manager 使用者對方案、管道和環境的存取。

## 簡介 {#introduction}

Cloud Manager有一組預先定義的角色，可管理各種Cloud Manager功能的存取權：

* 企業所有者
* 方案管理員
* 部署管理員
* 開發人員

自訂權限可讓使用者建立新的自訂權限設定檔並內含可設定的權限，以便限制 Cloud Manager 使用者對方案、管道和環境的存取。

>[!TIP]
>
>如需預先定義角色的詳細資訊，請參閱[角色型許可權](/help/requirements/role-based-permissions.md)。

## 使用自訂許可權 {#using}

建立和使用您自己的自訂許可權需要以下三個步驟：

1. [建立新的產品設定檔](#create)。
1. [指派自訂許可權給新的產品設定檔](#assign-permissions)。
1. [將使用者指派給新的產品設定檔](#assign-users)。

本節詳細說明這些步驟。 當您建立自己的自訂許可權時，可能會發現看到[條款](#terms)和[可設定的許可權](#configurable-permissions)區段很有用。

>[!NOTE]
>
>您必須在Admin Console中擁有產品管理員許可權，才能為Cloud Manager建立新設定檔及管理許可權。

### 建立新的產品設定檔 {#create}

首先，建立可指派自訂許可權的新產品設定檔。

1. 在 [my.cloudmanager.adobe.com](https://my.cloudmanager.adobe.com/) 登入 Cloud Manager。

1. 選取產品 **AEM Managed Services**。

1. 搜尋名稱符合模式`*-cloud-manager`的和執行個體，然後按一下以管理使用者和許可權。

1. 您被重新導向至Admin Console的&#x200B;**產品**&#x200B;標籤，您可以在其中管理Cloud Manager的使用者和許可權。 在Admin Console中，按一下&#x200B;**新增設定檔**。

![新設定檔按鈕](/help/assets/admin-console-new-profile.png)

1. 提供設定檔的一般詳細資訊。

   * **產品設定檔名稱**- 設定檔的說明性名稱
   * **顯示名稱** — 使用者介面（選項）中顯示的縮寫名稱
   * **說明** - 設定檔的資訊性描述，解釋用途 (選擇性)
   * **以電子郵件通知使用者** — 選取時，系統會在使用者加入此設定檔或從中移除時，以電子郵件通知使用者。

1. 按一下「**儲存**」。

新的產品設定檔已儲存並出現在 Admin Console 的產品設定檔清單。

### 將自訂許可權指派給新的產品設定檔 {#assign-permissions}

現在您有了新的產品設定檔，您可以將自訂權限指派至新的產品設定檔。

1. 在Admin Console中，按一下您剛建立的[新產品設定檔名稱](#create)。

1. 在開啟的視窗中，選取&#x200B;**權限**&#x200B;標籤可檢視可編輯的權限清單。

   ![可編輯的權限](/help/assets/permissions-tab.png)

1. 按一下&#x200B;**編輯**&#x200B;連結以取得編輯許可權。

1. **編輯權限**&#x200B;視窗隨即開啟。
   * 您在上一個步驟中選取的權限將在左側欄中呈現已選取狀態。
   * 標示為「**可用權限項目**」的中間欄包含此權限可指派的權限項目。
   * 標示為&#x200B;**包含的權限項目**&#x200B;的右側欄包含已指派的權限項目。

   ![編輯權限項目](/help/assets/edit-permission-items.png)

1. 按一下許可權專案旁的加號(`+`)圖示，將其新增至欄&#x200B;**包含的許可權專案**。 如有需要，請按一下許可權專案旁的`i`圖示以深入瞭解。

1. 在&#x200B;**可用許可權**&#x200B;欄的最上方，按一下&#x200B;**全部新增**&#x200B;以新增所有許可權。 同樣地，按一下&#x200B;**全部移除**&#x200B;以移除所有先前選取的許可權。

1. 當您完成定義新產品設定檔的許可權專案時，請按一下[儲存]。****

您的新產品設定檔現已與其自訂權限一起儲存。

### 將使用者指派給新的產品設定檔 {#assign-users}

現在您可以將使用者指派至您建立的內含自訂權限的新產品設定檔。

1. 在Admin Console中，按一下您剛指派自訂許可權的[新產品設定檔名稱](#assign-permissions)。

1. 在開啟的視窗中，選取&#x200B;**使用者**&#x200B;標籤。

1. 按一下&#x200B;**新增使用者**，並將使用者指派給具有自訂許可權的新產品設定檔。

請參閱檔案[管理企業使用者的產品設定檔](https://helpx.adobe.com/tw/enterprise/using/manage-product-profiles.html)的&#x200B;**將使用者和使用者群組新增到產品設定檔**，以取得有關如何使用此Admin Console的詳細資訊。

## 可設定的許可權 {#configurable-permissions}

以下權限可用於建立自訂設定檔。

| 權限 | 說明 |
|---|---|
| 方案存取 | 允許使用者存取方案 |
| 方案編輯 | 允許使用者編輯方案 |
| 管道建立 | 允許使用者建立新管道 |
| 管道刪除 | 允許使用者刪除管道 |
| 管道編輯 | 允許使用者編輯管道 |
| 生產部署核准/拒絕 | 允許使用者核准或拒絕生產部署步驟 |
| 管道執行取消 | 允許使用者取消管道執行 |
| 管道執行啟動 | 允許使用者啟動新的管道執行 |
| 覆寫/拒絕重要量度失敗 | 允許使用者覆寫/拒絕重要量度失敗 |
| 生產部署排程 | 允許使用者排程生產部署步驟 |
| 存放庫資訊存取 | 允許使用者存取存放庫資訊並產生存取密碼 |
| 存放庫建立 | 允許使用者建立新的 Git 存放庫 |
| 存放庫刪除 | 允許使用者刪除 Git 存放庫 |
| 存放庫編輯 | 允許使用者編輯Git存放庫 |
| 存放庫程式碼產生 | 允許使用者從原型版本產生專案 |
| 內容複製管理 | 允許使用者管理內容複製作業 |

### 組織層級許可權 {#organization-level}

組織層級的許可權一律會套用至組織內的所有計畫。

Cloud Manager中組織層級許可權的一個範例是&#x200B;**存放庫資訊存取**。 此許可權可讓使用者產生使用者名稱、密碼和存放庫URL，以便存取客戶專案及為其貢獻內容。 雖然組織中的所有存放庫的使用者名稱和密碼保持一致，但每個方案都有唯一的存放庫URL。

如需詳細資訊，請參閱[Source程式碼存放庫](/help/requirements/source-code-repository.md)。

## 詞彙 {#terms}

以下詞彙用於建立和管理自訂權限和預先定義的角色。

| 術語 | 說明 |
|---|---|
| 預先定義的權限 | 預先定義的角色，例如&#x200B;**企業所有者**、**部署管理員**&#x200B;等。 管理 Cloud Manager 的各種功能。如需預先定義角色的詳細資訊，請參閱[角色型許可權](/help/requirements/role-based-permissions.md)。 |
| 自訂權限 | Cloud Manager功能可讓使用者建立許可權設定檔，以定義角色來控管Cloud Manager的支援功能 |
| 權限設定檔 | 在Admin Console中建立，用於管理適用於屬於許可權設定檔的使用者之可設定許可權 |
| 可設定的權限 | Cloud Manager許可權可在許可權設定檔中設定 |
| 權限項目 | 可以套用權限的方案、環境或管道資源 |

許可權專案是指套用許可權的領域。 通常為下列其中一項。

| 權限項目類型 | 範例 | 說明 |
|---|---|---|
| 組織 | 組織：公司 A | 組織的所有適用資源。資源可以是方案、環境或管道。如果使用者為任何許可權新增組織，則該組織中的所有新資源也具有該許可權。 |
| 方案 | 方案 A | 方案的所有適用資源 |
| 環境 | 方案 A：環境 | 適用於特定環境 |
| 管道 | 方案 A：管道 | 適用於特定管道 |

## 限制 {#limitations}

使用自訂許可權時，請牢記以下限制：

* [只有一小組權限](#configurable-permissions)可用於建立自訂設定檔。
* 方案、環境、管道等資源。在 Cloud Manager 中建立，可能需要兩分鐘才會出現在 Admin Console 以進行權限設定。
* 在罕見的情況下，自訂許可權服務無法回應，預先定義的設定檔仍然可用，並且預先定義的設定檔中的使用者仍然具有適當的存取權。

## 常見問題 {#faq}

### 哪些權限設定檔是預先定義的權限設定檔？

* 企業所有者
* 方案管理員
* 部署管理員
* 開發人員

如需預先定義角色的詳細資訊，請參閱[角色型許可權](/help/requirements/role-based-permissions.md)。

### 引入自訂設定檔後，預先定義的權限設定檔會發生什麼情況？

預設產品設定檔和Cloud Manager角色仍與先前相同。

### 我可以編輯預先定義的權限設定檔嗎？

不可以，預設設定檔不可編輯。您無法新增或移除預設許可權設定檔的許可權。 您只能在預先定義的設定檔中新增或移除使用者。

### 由於現在可以使用自訂設定檔，我是否應該刪除預先定義的權限設定檔？

不可以從 Admin Console 刪除預先定義的權限設定檔。

### 我可以將使用者新增至多個權限設定檔嗎？

可以，一個使用者可以屬於多個設定檔，包括預先定義和自訂權限設定檔。將使用者指派給多個設定檔時，該使用者可以使用所有指派的許可權設定檔中的合併許可權。

### 如果使用者有權編輯環境/管道，但無權存取包含環境/管道的程式，會發生什麼情況？

在此案例中，如果使用者沒有包含環境或管道的&#x200B;**方案存取**&#x200B;許可權，則無法存取環境或管道。

### 如果我在同一個 IMS 組織中同時擁有 AEM as a Cloud Service 和 AMS 方案，會發生什麼事？我可以透過一個設定檔管理權限嗎？ {#ams-and-aemaacs}

為每種產品型別建立個別的設定檔。 也就是說，一個用於AEM as Cloud Service，另一個用於AdobeManaged Services或AMS。
